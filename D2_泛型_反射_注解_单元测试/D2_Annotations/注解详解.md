# Java 注解详解

---

## 一、什么是注解？

注解（Annotation）是 JDK 5 引入的一种**元数据**机制——它本身不影响代码逻辑，但可以在编译、部署、运行时被**读取和处理**，用来给代码添加额外信息。

---

### 1.1 注解 vs 注释

| | 注解（Annotation） | 注释（Comment） |
|------|------|------|
| 写法 | `@Xxx` | `//` 或 `/* */` |
| 能否被程序读取 | ✅ 编译/运行时可以读取 | ❌ 编译器直接忽略 |
| 能否影响代码行为 | ✅ 配合工具/框架可以 | ❌ 纯粹给人看 |
| 典型用途 | `@Override`、`@Test`、`@Autowired` | 解释代码逻辑 |

### 1.2 注解能做什么？

| 用途 | 示例 |
|------|------|
| **编译检查** | `@Override` 让编译器检查是否真正重写了父类方法 |
| **替代配置文件** | `@WebServlet("/login")` 替代 web.xml |
| **代码生成** | Lombok 的 `@Getter` / `@Setter` 自动生成方法 |
| **运行时反射处理** | Spring 的 `@Autowired` 自动注入依赖 |
| **单元测试标记** | JUnit 的 `@Test` 标记测试方法 |

## 二、内置注解

Java 自带了一些开箱即用的注解：

---

### 2.1 @Override — 重写检查

告诉编译器这个方法是重写父类/接口的方法，如果签名写错了，编译器就报错。

```java
class Parent {
    public void sayHello() { }
}

class Child extends Parent {
    @Override
    public void sayHello() { }  // ✅ 正确重写

    // @Override
    // public void sayHello(String name) { }  // ❌ 编译错误！签名不匹配
}
```

> **最佳实践：** 凡是重写的方法，一律加上 `@Override`。

---

### 2.2 @Deprecated — 标记过时

标记某个方法/类/字段已过时，不推荐使用，调用时会显示删除线。

```java
public class OldUtils {

    @Deprecated
    public static void oldMethod() {
        System.out.println("这个方法已过时，请使用 newMethod()");
    }

    public static void newMethod() {
        System.out.println("新方法");
    }
}
```

JDK 9 新增了两个属性：

```java
@Deprecated(since = "2.0", forRemoval = true)
//          ↑ 从哪个版本开始过时   ↑ true = 未来会删除
public void legacyCode() { }
```

---

### 2.3 @SuppressWarnings — 抑制编译器警告

告诉编译器"我知道这里有问题，别报警告了"。

```java
@SuppressWarnings("unchecked")   // 抑制"未检查类型转换"警告
public void legacyMethod() {
    List rawList = new ArrayList();        // 没有泛型，会有警告
    List<String> list = (List<String>) rawList;  // 不安全的类型转换
}

@SuppressWarnings({"unchecked", "deprecation"})  // 同时抑制多个
public void mixedMethod() { ... }
```

常见参数值：

| 参数 | 抑制的警告类型 |
|------|--------------|
| `"unchecked"` | 未检查的类型转换 |
| `"deprecation"` | 使用了过时的 API |
| `"rawtypes"` | 使用原始类型（如 `List` 而非 `List<String>`） |
| `"unused"` | 未使用的变量/方法 |
| `"all"` | 所有警告 |

---

### 2.4 @FunctionalInterface — 函数式接口检查

确保接口只有一个抽象方法（否则编译报错），用于 Lambda 表达式。

```java
@FunctionalInterface
public interface Calculator {
    int calculate(int a, int b);  // 只有一个抽象方法 ✅

    // int other();  // ❌ 如果加一个抽象方法，编译报错

    default void print() { }      // ✅ default 方法可以有多个
}
```

---

### 2.5 @SafeVarargs — 安全可变参数（了解）

抑制"参数化可变参数类型"的警告，通常用在泛型工具方法上。

```java
@SafeVarargs
public static <T> List<T> asList(T... items) {
    return Arrays.asList(items);
}
```

## 三、元注解（注解的注解）

元注解是**作用在注解定义上的注解**，用来控制注解的行为。

---

### 3.1 @Target — 指定注解能用在哪儿

```java
@Target(ElementType.METHOD)     // 只能用在方法上
public @interface Test { }

@Target({ElementType.TYPE, ElementType.METHOD})  // 可以用在类和方法上
public @interface MyAnnotation { }
```

`ElementType` 可选值：

| 值 | 适用范围 |
|----|----------|
| `TYPE` | 类、接口、枚举、注解 |
| `FIELD` | 成员变量（包括枚举常量） |
| `METHOD` | 方法 |
| `PARAMETER` | 方法参数 |
| `CONSTRUCTOR` | 构造方法 |
| `LOCAL_VARIABLE` | 局部变量 |
| `ANNOTATION_TYPE` | 注解类型 |
| `PACKAGE` | 包 |
| `TYPE_PARAMETER` | 类型参数（JDK 8+：`<@NonNull T>`） |
| `TYPE_USE` | 任何类型使用处（JDK 8+） |

---

### 3.2 @Retention — 指定注解保留到什么时候

```java
@Retention(RetentionPolicy.RUNTIME)  // 保留到运行时，可被反射读取
public @interface RuntimeAnnotation { }

@Retention(RetentionPolicy.SOURCE)   // 只在源码中存在，编译器丢弃
public @interface SourceAnnotation { }
```

| 保留策略 | 说明 | 典型用例 |
|----------|------|----------|
| `SOURCE` | 只存在于源码中，编译时丢弃 | `@Override`、Lombok 的 `@Getter` |
| `CLASS` | 编译到 `.class` 文件中，但运行时 JVM 不加载（**默认值**） | 很少直接用 |
| `RUNTIME` | 保留到运行时，可被反射读取 | `@Test`、`@Autowired`、`@Deprecated` |

---

### 3.3 @Documented — 包含在 Javadoc 中

```java
@Documented
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation { }
```

加了 `@Documented` 的注解，用 `javadoc` 生成文档时会出现在文档中。

---

### 3.4 @Inherited — 允许子类继承

```java
@Inherited
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface InheritableAnnotation { }

@InheritableAnnotation
class Parent { }

class Child extends Parent { }
// Child 也会被认为是 @InheritableAnnotation（因为 @Inherited）
```

> `@Inherited` 只在**类继承**时有效，接口实现不会继承。

---

### 3.5 @Repeatable — 可重复注解（JDK 8+）

同一个位置可以多次使用同一个注解：

```java
// 1. 定义容器注解
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface Roles {
    Role[] value();  // 容器注解中的 value 返回子注解数组
}

// 2. 定义可重复注解，用 @Repeatable 指向容器
@Repeatable(Roles.class)
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface Role {
    String value();
}

// 3. 使用：可以写多个 @Role
@Role("admin")
@Role("user")
public class UserService { }
```

## 四、自定义注解

---

### 4.1 基本语法

```java
public @interface 注解名 {
    // 属性（看起来像方法，没有参数）
    类型 属性名() default 默认值;
}

// 使用
@注解名(属性名 = 值)
```

### 4.2 定义第一个自定义注解

```java
// 定义
@Target(ElementType.TYPE)         // 用在类上
@Retention(RetentionPolicy.RUNTIME)  // 运行时保留
@Documented
public @interface Table {
    String value();      // 表名（必填，没有 default）
    String engine() default "InnoDB";  // 存储引擎（可选，有默认值）
    String charset() default "utf8mb4"; // 字符集（可选）
}

// 使用
@Table(value = "t_user", engine = "MyISAM")
public class User { ... }

@Table("t_order")  // 如果只有 value 属性，可以省略 "value ="
public class Order { ... }
```

---

### 4.3 属性的数据类型

注解属性可以是以下类型：

| 类型 | 示例 |
|------|------|
| 基本数据类型 | `int count() default 0;` |
| `String` | `String name() default "";` |
| `Class` | `Class<?> type() default Object.class;` |
| 枚举 | `ElementType target() default ElementType.TYPE;` |
| 注解 | `AnotherAnnotation nested() default @AnotherAnnotation;` |
| 以上类型的数组 | `int[] ids() default {};` |

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiOperation {
    String value();                              // 描述
    int timeout() default 3000;                  // 超时（毫秒）
    Class<?> returnType() default void.class;    // 返回类型
    HttpMethod method() default HttpMethod.GET;  // HTTP 请求方式
}

// 枚举
enum HttpMethod { GET, POST, PUT, DELETE }
```

---

### 4.4 属性命名约定

```java
// ✅ value —— 特殊属性名，使用时可以省略 "value = "
@Table("t_user")                    // 等价于 @Table(value = "t_user")

// ✅ 其他属性名 —— 必须显式指定
@Table(value = "t_user", engine = "MyISAM")   // 不能写成 @Table("t_user", "MyISAM")

// ✅ 数组属性 —— 单元素可以省略大括号
@Roles({"admin"})                   // 数组标准写法
@Roles("admin")                     // 单元素时可以简写
```

## 五、运行时读取注解（反射 + 注解）

---

### 5.1 读取类上的注解

```java
// 获取类上的注解
Class<User> clazz = User.class;

// 判断是否有某个注解
if (clazz.isAnnotationPresent(Table.class)) {
    Table table = clazz.getAnnotation(Table.class);
    System.out.println(table.value());    // "t_user"
    System.out.println(table.engine());   // "InnoDB"
    System.out.println(table.charset());  // "utf8mb4"
}
```

### 5.2 读取字段上的注解

```java
public class User {
    @Column(name = "user_name", length = 50)
    private String name;

    @Column(name = "user_age")
    private int age;
}

// 遍历所有字段，读取 @Column 注解
for (Field field : User.class.getDeclaredFields()) {
    if (field.isAnnotationPresent(Column.class)) {
        Column col = field.getAnnotation(Column.class);
        String columnName = col.name();
        int length = col.length();
        System.out.println(field.getName() + " → " + columnName);
    }
}
```

### 5.3 读取方法上的注解

```java
public class Service {
    @LogExecution(level = "INFO")
    public void processOrder() { ... }
}

// 获取方法上的注解
Method method = Service.class.getMethod("processOrder");
if (method.isAnnotationPresent(LogExecution.class)) {
    LogExecution log = method.getAnnotation(LogExecution.class);
    System.out.println("日志级别：" + log.level());
}
```

### 5.4 完整示例：自动建表

```java
// === 注解定义 ===
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@interface Table {
    String value();
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@interface Column {
    String name();
    String type() default "VARCHAR(255)";
    boolean nullable() default true;
}

// === 使用 ===
@Table("t_user")
class User {
    @Column(name = "id", type = "INT", nullable = false)
    private Long id;

    @Column(name = "user_name", type = "VARCHAR(50)")
    private String name;

    @Column(name = "age", type = "INT")
    private int age;
}

// === 注解处理器：生成建表 SQL ===
public class TableGenerator {
    public static String generateSQL(Class<?> clazz) {
        Table table = clazz.getAnnotation(Table.class);
        StringBuilder sb = new StringBuilder("CREATE TABLE " + table.value() + " (\n");

        Field[] fields = clazz.getDeclaredFields();
        for (int i = 0; i < fields.length; i++) {
            Column col = fields[i].getAnnotation(Column.class);
            if (col != null) {
                sb.append("  ").append(col.name()).append(" ").append(col.type());
                if (!col.nullable()) sb.append(" NOT NULL");
                if (i < fields.length - 1) sb.append(",");
                sb.append("\n");
            }
        }
        sb.append(");");
        return sb.toString();
    }
}

// 调用：
// String sql = TableGenerator.generateSQL(User.class);
// 输出：
// CREATE TABLE t_user (
//   id INT NOT NULL,
//   user_name VARCHAR(50),
//   age INT
// );
```

## 六、注解处理器（编译时）

---

### 6.1 两种处理方式

| 处理方式 | 时机 | 典型代表 |
|----------|------|----------|
| **反射处理** | 运行时 | Spring `@Autowired`、JUnit `@Test` |
| **编译时处理（APT）** | 编译期 | Lombok `@Getter`、MapStruct `@Mapper` |

### 6.2 编译时注解处理器概念

编译时处理器实现了 `javax.annotation.processing.Processor` 接口（一般继承 `AbstractProcessor`），能在编译期间扫描注解，生成新的 `.java` 源文件或 `.class` 文件。

```java
// 概念示例（简化）
@SupportedAnnotationTypes("com.example.GenerateBuilder")
@SupportedSourceVersion(SourceVersion.RELEASE_17)
public class BuilderProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                           RoundEnvironment roundEnv) {
        // 扫描所有 @GenerateBuilder 注解的类
        // 为每个类生成对应的 Builder 类源码文件
        return true;
    }
}
```

> 完整的 APT 处理器涉及 `JavaPoet` 等代码生成库，属于高级话题，此处了解概念即可。

## 七、注解 vs 反射 vs 配置文件的取舍

| 方案 | 优点 | 缺点 |
|------|------|------|
| **注解** | 代码内聚，一眼看到配置 | 修改需要重新编译 |
| **XML / 配置文件** | 修改无需重编译，集中管理 | 啰嗦，与代码割裂 |
| **纯反射** | 完全动态 | 没有类型安全，出错运行时才知道 |

**主流趋势：** Spring Boot 等现代框架大量使用注解（`@Bean`、`@RequestMapping`），逐步替代 XML。

## 八、命名规范

```java
// ✅ 类名以 Annotation 结尾不是硬要求，但一般直接描述用途
@Test          // 标记测试方法
@Override      // 标记重写
@Deprecated    // 标记过时
@Autowired     // 标记自动注入

// ✅ 自定义注解建议用名词或动词
@Table         // 描述一个数据库表
@Column        // 描述一个数据库列
@LogExecution  // 标记需要记录日志
@Retry         // 标记需要重试
```

## 九、关键要点速查

| 概念 | 一句话说明 |
|------|-----------|
| **注解** | 给代码附加元数据，可在编译/运行时被处理 |
| **@Override** | 编译检查是否真正重写了父类方法 |
| **@Deprecated** | 标记过时 API，调用时有删除线警告 |
| **@SuppressWarnings** | 抑制编译器警告 |
| **@FunctionalInterface** | 确保接口只有一个抽象方法 |
| **@Target** | 元注解：指定注解能用在哪儿（类/方法/字段等） |
| **@Retention** | 元注解：指定注解保留到什么时候（源码/字节码/运行时） |
| **@Repeatable** | 元注解：同一个位置可多次使用该注解（JDK 8+） |
| **@Documented** | 元注解：让注解出现在 Javadoc 中 |
| **@Inherited** | 元注解：让子类继承父类的注解 |
| **自定义注解** | `@interface` 定义，属性如 `String value() default ""` |
| **value 属性** | 特殊属性名，使用时可以省略 `value =` |
| **isAnnotationPresent** | 反射判断是否存在某个注解 |
| **getAnnotation** | 反射读取注解对象 |
