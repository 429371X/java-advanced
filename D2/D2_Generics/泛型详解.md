# Java 泛型详解

---

## 第一部分：泛型 (Generics)

Java 泛型是 JDK 5 引入的特性，允许在定义类、接口和方法时使用类型参数，从而实现**类型安全**和**代码复用**。

---

### 一、类型擦除 (Type Erasure)

#### 1.1 什么是类型擦除？

Java 的泛型是**编译时**特性——编译器在编译期间会"擦除"泛型类型信息，将类型参数替换为它们的**边界类型**（默认是 `Object`），并在需要时插入强制类型转换。

#### 1.2 擦除规则

| 类型参数形式 | 擦除后的替换类型 |
|-------------|---------------|
| `<T>` (无边界) | `Object` |
| `<T extends Number>` | `Number` |
| `<T extends Comparable<T>>` | `Comparable` |

#### 1.3 示例

```java
// 编译前
public class Box<T> {
    private T value;
    public T get() { return value; }
    public void set(T value) { this.value = value; }
}

// 编译后（等价于）
public class Box {
    private Object value;          // T → Object
    public Object get() { return value; }
    public void set(Object value) { this.value = value; }
}
```

```java
// 有边界的类型参数
public class NumBox<T extends Number> {
    private T value;
    public T get() { return value; }
}

// 编译后（等价于）
public class NumBox {
    private Number value;          // T → Number（边界类型）
    public Number get() { return value; }
}
```

#### 1.4 类型擦除的后果

1. **无法用基本数据类型实例化泛型**：不能写 `List<int>`，只能写 `List<Integer>`
2. **无法获取泛型的运行时类型**：`new ArrayList<String>() instanceof ArrayList<?>` 返回 `true`，但无法区分 `ArrayList<String>` 和 `ArrayList<Integer>`
3. **不能创建泛型数组**：`new T[10]` 在运行时不知道 T 是什么
4. **桥接方法**：编译器自动生成桥接方法来保持多态性

```java
// 桥接方法示例
class Parent<T> {
    public void say(T t) { }
}
class Child extends Parent<String> {
    @Override
    public void say(String s) { }   // 编译器生成桥接方法 say(Object)
}
```

### 二、通配符 (Wildcards)

通配符 `?` 表示**未知类型**，用于使泛型代码更灵活。

#### 2.1 无界通配符 `<?>`

表示可以接收任意类型，但不能添加元素（除了 `null`）：

```java
public void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
    // list.add("abc");  // ❌ 编译错误：无法确定类型
}
```

#### 2.2 上界通配符 `<? extends T>` — 生产者 (Producer)

表示类型是 `T` 或 `T` 的子类，**只能读，不能写**：

```java
// 可以接收 List<Integer>、List<Double>、List<Number> 等
public double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {   // ✅ 可以读（安全，元素至少是 Number）
        total += n.doubleValue();
    }
    // list.add(10);          // ❌ 编译错误：不能写入
    return total;
}
```

**PECS 原则 — Producer Extends**：当你只从结构中**读取**数据时，使用 `extends`。

#### 2.3 下界通配符 `<? super T>` — 消费者 (Consumer)

表示类型是 `T` 或 `T` 的父类，**只能写，读受限**：

```java
// 可以向 List<Integer>、List<Number>、List<Object> 中添加 Integer
public void addNumbers(List<? super Integer> list) {
    list.add(1);              // ✅ 可以写（安全，容器至少能装 Integer）
    list.add(2);
    list.add(3);
    // Integer x = list.get(0);  // ❌ 编译错误：读取返回 Object
}
```

**PECS 原则 — Consumer Super**：当你只向结构中**写入**数据时，使用 `super`。

#### 2.4 PECS 总结

| 场景 | 使用 | 可读 | 可写 |
|------|------|------|------|
| 只读 | `<? extends T>` | ✅ 读为 T | ❌ |
| 只写 | `<? super T>` | ⚠️ 读为 Object | ✅ 写 T/T的子类 |
| 读写 | 直接用 `<T>` | ✅ | ✅ |

```java
// PECS 典型应用：Collections.copy()
public static <T> void copy(List<? super T> dest,   // 消费者 — 写入目标
                            List<? extends T> src) { /* ... */ }  // 生产者 — 读取源
```

### 三、泛型方法 / 类 / 接口

#### 3.1 泛型类 (Generic Class)

在类名后声明类型参数，整个类内部都可以使用：

```java
// 泛型类：一个简单的键值对容器
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey()   { return key; }
    public V getValue() { return value; }
}

// 使用
Pair<String, Integer> pair = new Pair<>("age", 25);  // JDK 7+ 钻石语法
```

#### 3.2 泛型接口 (Generic Interface)

在接口名后声明类型参数：

```java
// 泛型接口：数据访问层
public interface Repository<T> {
    void save(T entity);
    T findById(Long id);
    List<T> findAll();
}

// 实现泛型接口（指定具体类型）
public class UserRepository implements Repository<User> {
    @Override
    public void save(User entity) { /* ... */ }
    @Override
    public User findById(Long id) { /* ... */ }
    @Override
    public List<User> findAll() { /* ... */ }
}

// 实现泛型接口（保持泛型）
public class GenericRepository<T> implements Repository<T> {
    @Override
    public void save(T entity) { /* ... */ }
    // ...
}
```

#### 3.3 泛型方法 (Generic Method)

在方法返回值前用 `<T>` 声明类型参数——**即使所在的类不是泛型类**：

```java
public class Utils {

    // 泛型方法：交换数组中两个元素
    public static <T> void swap(T[] array, int i, int j) {
        T temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }

    // 泛型方法：查找数组中第一个匹配的元素
    public static <T> int indexOf(T[] array, T target) {
        for (int i = 0; i < array.length; i++) {
            if (array[i].equals(target)) return i;
        }
        return -1;
    }

    // 有边界约束的泛型方法
    public static <T extends Comparable<T>> T max(T a, T b) {
        return a.compareTo(b) > 0 ? a : b;
    }
}

// 调用
public class Main {
    public static void main(String[] args) {
        String[] names = {"A", "B", "C"};
        Utils.swap(names, 0, 2);
        int index = Utils.indexOf(names, "B");
        String bigger = Utils.max("apple", "banana");  // "banana"
    }
}
```

#### 3.4 多重边界

```java
// T 必须是 Number 的子类，且实现了 Comparable 接口
public static <T extends Number & Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) > 0 ? a : b;
}
// ⚠️ 注意：类边界放在前面，接口边界放在后面（最多一个类边界）
```

## 关键要点速查

| 概念 | 一句话说明 |
|------|-----------|
| **类型擦除** | 编译时泛型类型信息被替换为边界类型（默认 Object） |
| **<? extends T>** | 上界通配符，**生产者**—可读不可写 |
| **<? super T>** | 下界通配符，**消费者**—可写不可读 |
| **泛型方法** | 在返回值前声明 `<T>`，即使类本身不是泛型 |
