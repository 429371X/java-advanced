# Java 反射详解

---

## 一、什么是反射？

反射（Reflection）是 Java 在**运行时**动态获取类的信息、操作类的属性和方法的能力。它让程序能够在编译时不知道具体类名的情况下，在运行时检查并操作任意对象。

---

### 1.1 反射能做什么？

| 能力 | 说明 |
|------|------|
| **获取 Class 对象** | 在运行时拿到任意类的 `Class` 对象 |
| **获取构造方法** | 获取并调用构造器，动态创建对象 |
| **获取成员变量** | 获取字段信息，读写字段值（包括 `private`） |
| **获取成员方法** | 获取方法信息，动态调用方法（包括 `private`） |
| **获取注解信息** | 读取类/方法/字段上的注解 |
| **获取泛型信息** | 读取方法参数和返回值的泛型类型 |

### 1.2 反射的核心入口：Class 对象

JVM 每加载一个类，就会在内存中为其创建一个唯一的 `Class` 对象。这个对象包含了这个类的**所有信息**，反射的一切操作都从获取 `Class` 对象开始。

---

## 二、获取 Class 对象的三种方式

```java
// 方式1：类名.class（编译时已知类名，最常用）
Class<String> clazz1 = String.class;

// 方式2：对象.getClass()（已有对象实例）
String str = "hello";
Class<?> clazz2 = str.getClass();

// 方式3：Class.forName("全限定类名")（运行时动态加载，常用于配置文件）
Class<?> clazz3 = Class.forName("java.lang.String");
```

| 方式 | 时机 | 适用场景 |
|------|------|----------|
| `类名.class` | 编译时 | 代码中明确知道要操作的类 |
| `对象.getClass()` | 运行时 | 已有对象实例 |
| `Class.forName("...")` | 运行时 | 类名来自配置文件、数据库等，JDBC 驱动加载的经典用法 |

> `Class.forName("xxx")` 会触发类的**静态初始化**（执行 `static` 代码块），而 `类名.class` 不会。

## 三、获取构造方法（Constructor）

---

### 3.1 获取 Constructor 对象

```java
Class<Person> clazz = Person.class;

// 获取所有 public 构造方法
Constructor<?>[] constructors = clazz.getConstructors();

// 获取所有构造方法（包括 private）
Constructor<?>[] allConstructors = clazz.getDeclaredConstructors();

// 获取指定参数的 public 构造方法
Constructor<Person> c1 = clazz.getConstructor();                // 无参
Constructor<Person> c2 = clazz.getConstructor(String.class, int.class);  // (String, int)

// 获取指定参数的构造方法（包括 private）
Constructor<Person> c3 = clazz.getDeclaredConstructor(String.class);
```

### 3.2 调用构造方法创建对象

```java
// 调用 public 无参构造
Constructor<Person> c1 = clazz.getConstructor();
Person p1 = c1.newInstance();

// 调用 public 有参构造
Constructor<Person> c2 = clazz.getConstructor(String.class, int.class);
Person p2 = c2.newInstance("张三", 25);

// 调用 private 构造（需要先暴力反射）
Constructor<Person> c3 = clazz.getDeclaredConstructor(String.class);
c3.setAccessible(true);  // ⚡ 取消访问检查
Person p3 = c3.newInstance("李四");
```

## 四、获取成员变量（Field）

---

### 4.1 获取 Field 对象

```java
Class<Person> clazz = Person.class;

// 获取所有 public 字段（包括父类继承的）
Field[] fields = clazz.getFields();

// 获取本类所有字段（包括 private，不包括父类）
Field[] allFields = clazz.getDeclaredFields();

// 获取指定名称的 public 字段
Field nameField = clazz.getField("name");

// 获取指定名称的字段（包括 private）
Field ageField = clazz.getDeclaredField("age");
```

### 4.2 读写字段值

```java
public class Person {
    public String name;
    private int age;
}

// --- 读写 public 字段 ---
Person p = new Person();
Field nameField = Person.class.getField("name");
nameField.set(p, "张三");              // 设置值
String name = (String) nameField.get(p); // 读取值

// --- 读写 private 字段 ---
Field ageField = Person.class.getDeclaredField("age");
ageField.setAccessible(true);  // ⚡ 暴力反射
ageField.set(p, 25);           // 设置值
int age = (int) ageField.get(p); // 读取值
```

## 五、获取成员方法（Method）

---

### 5.1 获取 Method 对象

```java
Class<Person> clazz = Person.class;

// 获取所有 public 方法（包括父类继承的，如 Object 的方法）
Method[] methods = clazz.getMethods();

// 获取本类所有方法（包括 private，不包括父类）
Method[] allMethods = clazz.getDeclaredMethods();

// 获取指定名称和参数的 public 方法
Method m1 = clazz.getMethod("sayHello");                         // 无参
Method m2 = clazz.getMethod("sayHello", String.class);           // (String)

// 获取指定名称和参数的方法（包括 private）
Method m3 = clazz.getDeclaredMethod("secretMethod");
```

### 5.2 调用方法

```java
public class Person {
    public void sayHello(String name) {
        System.out.println("Hello, " + name);
    }
    private String getSecret() {
        return "secret info";
    }
}

// --- 调用 public 方法 ---
Person p = new Person();
Method method = Person.class.getMethod("sayHello", String.class);
method.invoke(p, "张三");  // 输出：Hello, 张三

// --- 调用 private 方法 ---
Method secretMethod = Person.class.getDeclaredMethod("getSecret");
secretMethod.setAccessible(true);  // ⚡ 暴力反射
String result = (String) secretMethod.invoke(p);
System.out.println(result);  // 输出：secret info

// --- 调用静态方法（invoke 第一个参数传 null） ---
Method staticMethod = Person.class.getMethod("staticMethod");
staticMethod.invoke(null);
```

### 5.3 invoke 方法签名

```java
Object invoke(Object obj, Object... args)
//             ↑          ↑
//        目标对象    方法参数（可变参）
//        静态方法传 null
```

## 六、获取类的基本信息

```java
Class<?> clazz = Person.class;

// 基本信息
String className = clazz.getName();                // "com.example.Person"（全限定名）
String simpleName = clazz.getSimpleName();         // "Person"
Package pkg = clazz.getPackage();                  // 包信息
int modifiers = clazz.getModifiers();              // 修饰符（public=1, abstract=1024...）

// 判断修饰符
boolean isPublic = Modifier.isPublic(modifiers);
boolean isAbstract = Modifier.isAbstract(modifiers);

// 父类和接口
Class<?> superClass = clazz.getSuperclass();       // 父类
Class<?>[] interfaces = clazz.getInterfaces();     // 实现的接口

// 注解
Annotation[] annotations = clazz.getAnnotations(); // 获取所有注解
MyAnnotation ann = clazz.getAnnotation(MyAnnotation.class); // 获取指定注解
```

## 七、setAccessible — 暴力反射

---

### 7.1 什么是 setAccessible？

`Field`、`Method`、`Constructor` 都继承自 `AccessibleObject`，它有一个 `accessible` 标志位。当访问 `private` 成员时：

- 默认 `accessible = false` → JVM 会进行访问权限检查 → 抛 `IllegalAccessException`
- 调用 `setAccessible(true)` → 跳过检查 → 可以访问 `private` 成员

```java
Field privateField = clazz.getDeclaredField("secret");
privateField.setAccessible(true);  // ⚡ 突破封装
String secret = (String) privateField.get(obj);
```

### 7.2 安全限制

> 高版本 JDK（9+）模块系统可能会限制对 JDK 内部类的暴力反射，应用代码之间的反射不受影响。Java 17+ 需要添加 `--add-opens` 参数才能反射 JDK 内部 API。

## 八、反射的实际应用场景

---

| 场景 | 说明 |
|------|------|
| **框架底层** | Spring 的 IoC 容器通过反射创建 Bean、注入依赖 |
| **ORM 框架** | MyBatis/Hibernate 通过反射把数据库行映射成 Java 对象 |
| **JDBC 驱动加载** | `Class.forName("com.mysql.cj.jdbc.Driver")` |
| **单元测试** | JUnit 通过反射发现并运行带 `@Test` 注解的方法 |
| **序列化/反序列化** | JSON 库（Jackson/Gson）通过反射读写对象字段 |
| **动态代理** | 运行时生成代理类，结合反射调用目标方法 |
| **IDE 智能提示** | 通过反射读取类的方法列表、参数类型 |
| **配置文件驱动** | 在配置文件中写类名和方法名，运行时动态加载执行 |

---

### 8.1 经典示例：简单的 IoC 容器

```java
// 配置文件内容：className=com.example.UserService
public static Object createBean(String className) throws Exception {
    // 1. 加载类
    Class<?> clazz = Class.forName(className);

    // 2. 通过无参构造创建实例
    Constructor<?> constructor = clazz.getConstructor();
    Object instance = constructor.newInstance();

    // 3. 注入依赖（简化演示：遍历所有字段，有 @Autowired 就注入）
    for (Field field : clazz.getDeclaredFields()) {
        if (field.isAnnotationPresent(Autowired.class)) {
            field.setAccessible(true);
            Object dependency = createBean(field.getType().getName());
            field.set(instance, dependency);
        }
    }

    return instance;
}
```

---

### 8.2 经典示例：读取配置文件创建任意对象

```java
// properties 文件：person=com.example.Person
public static Object getBean(String key) throws Exception {
    Properties props = new Properties();
    props.load(new FileReader("config.properties"));
    String className = props.getProperty(key);
    return Class.forName(className).getConstructor().newInstance();
}
```

## 九、反射性能开销

反射比直接调用慢，主要体现在：

| 开销来源 | 说明 |
|----------|------|
| **访问检查** | 每次访问都要检查安全权限 |
| **类型包装** | 参数和返回值需要装箱/拆箱 |
| **方法查找** | 需要从元数据中查找方法信息 |

**优化建议：**

```java
// ❌ 每次都查找 Method（慢）
for (int i = 0; i < 10000; i++) {
    Method m = clazz.getMethod("sayHello");
    m.invoke(obj);
}

// ✅ Method 对象缓存起来复用（快很多）
Method m = clazz.getMethod("sayHello");
for (int i = 0; i < 10000; i++) {
    m.invoke(obj);
}

// ✅ 进一步优化：MethodHandle（JDK 7+）或 LambdaMetaFactory（JDK 8+）
```

## 十、常用反射 API 速查

### 10.1 操作流程总结

```
获取 Class 对象
    │
    ├─→ getConstructor()     → Constructor.newInstance()  → 创建对象
    ├─→ getField()           → Field.get() / Field.set()  → 读写字段
    ├─→ getMethod()          → Method.invoke()            → 调用方法
    └─→ getAnnotation()      → 读取注解信息
```

### 10.2 方法命名规律

| 前缀 | 范围 | 示例 |
|------|------|------|
| `getXxx()` | 只获取 `public` 成员 | `getField()`、`getMethod()`、`getConstructor()` |
| `getDeclaredXxx()` | 获取**所有**成员（含 `private`） | `getDeclaredField()`、`getDeclaredMethod()` |
| 单数 | 获取**指定名称**的一个 | `getField("name")`、`getMethod("say")` |
| 复数 | 获取**全部**组成的数组 | `getFields()`、`getMethods()` |

## 十一、各类型常用 API 一览

### 11.1 Constructor 常用方法

| 方法 | 说明 |
|------|------|

| `get(Object obj)` | 读取 obj 的该字段值 |
| `set(Object obj, Object value)` | 设置 obj 的该字段值 |
| `setAccessible(true)` | 取消访问检查 |

### 11.3 Method 常用方法

| 方法 | 说明 |
|------|------|
| `getName()` | 获取方法名 |
| `getReturnType()` | 获取返回值类型（`Class<?>`） |
| `getParameterTypes()` | 获取参数类型数组 |
| `invoke(Object obj, Object... args)` | 调用方法 |
| `setAccessible(true)` | 取消访问检查 |

## 十二、关键要点速查

| 概念 | 一句话说明 |
|------|-----------|
| **Class 对象** | JVM 中类的元数据，反射操作入口 |
| **三种获取方式** | `类名.class` / `对象.getClass()` / `Class.forName("全限定名")` |
| **Constructor** | 反射构造方法，`getConstructor()` → `newInstance()` 创建对象 |
| **Field** | 反射字段，`getDeclaredField()` → `get()` / `set()` 读写值 |
| **Method** | 反射方法，`getDeclaredMethod()` → `invoke()` 调用 |
| **setAccessible(true)** | 暴力反射，绕过 `private` 访问限制 |
| **getXxx vs getDeclaredXxx** | 前者只拿 `public`，后者拿全部（含 `private`） |
| **反射应用** | Spring IoC、ORM 映射、JSON 序列化、JUnit、JDBC 驱动 |
| **性能注意** | 反射比直接调用慢，Method/Field 对象应缓存复用 |
