# Java 集合框架基本结构

---

> 本文只包括常见的集合框架

---

## 一、概述

Java 集合框架主要分为两大类：

1. **单列集合（Collection）**：一次存储一个元素（单个值）
2. **双列集合（Map）**：一次存储一对元素（键值对 key-value）

---

## 二、单列集合（Collection 接口）

**特点：** 单列集合的顶层接口是 `Collection`，继承自 `Iterable` 接口，所有单列集合都可以使用增强 for 循环或迭代器遍历。

```
Collection（接口）
├── List（接口）—— 有序、可重复、有索引
└── Set（接口） —— 无序、不可重复、无索引
```

---

### 2.1 List 集合 —— 有序、可重复、有索引

**特点：** 元素存入和取出的顺序一致，允许存储重复元素，通过索引可以精确操作元素。

#### ArrayList

- 底层基于**数组**实现，查询快，增删慢
- 线程不安全，效率高
- 默认初始容量 **10**，扩容为原来的 **1.5 倍**

#### LinkedList

- 底层基于**双向链表**实现，查询慢，增删快
- 线程不安全，效率高
- 实现了 `List` 和 `Deque` 接口，可作队列/栈使用

---

### 2.2 Set 集合 —— 无序、不可重复、无索引

**特点：** 元素存取顺序可能不一致，不能存储重复元素，没有索引，不能使用普通 for 循环遍历。

#### HashSet

- 底层基于 `HashMap`（哈希表）实现
- 存取无序，不允许重复
- 默认初始容量 **16**，加载因子 **0.75**
- 依赖 `hashCode()` 和 `equals()` 方法保证唯一性

#### LinkedHashSet

- 继承自 `HashSet`，底层基于 `LinkedHashMap`
- 存取有序（通过双向链表维护插入顺序）
- 不允许重复，性能略低于 `HashSet`

#### TreeSet

- 底层基于 `TreeMap`（红黑树）实现
- 元素会按照自然顺序或比较器规则自动排序
- 不允许重复，不允许 `null`（取决于比较器）
- 存储的元素必须实现 `Comparable` 接口

---

## 三、双列集合（Map 接口）

**特点：** 双列集合的顶层接口是 `Map`，存储的是键值对（key-value）。key 是唯一的（不可重复），value 可以重复。每个 key 只能对应一个 value。**Map 接口独立于 Collection，不继承 Collection。**

```
Map（接口）
├── HashMap
├── LinkedHashMap
└── TreeMap
```

---

### 3.1 各细分集合

#### HashMap

- 底层基于哈希表（**数组 + 链表 + 红黑树**，JDK 8+）
- key 无序，不允许重复；key 和 value 都可以为 `null`
- 线程不安全，效率高
- 默认初始容量 **16**，加载因子 **0.75**
- 当链表长度 > 8 且数组长度 >= 64 时，链表转红黑树

#### LinkedHashMap

- 继承自 `HashMap`，通过双向链表维护键值对的插入顺序
- key 有序（插入顺序或访问顺序）
- 线程不安全
- 适合需要有序遍历的场景（如 **LRU 缓存**）

#### TreeMap

- 底层基于**红黑树**实现
- key 会按照自然顺序或比较器规则自动排序
- key 不允许 `null`（取决于比较器），value 可以为 `null`
- 线程不安全
- key 必须实现 `Comparable` 接口

---

## 四、对比总结

| 维度 | List | Set | Map |
|------|------|-----|-----|
| 是否有序 | 有序（索引） | 通常无序 | 取决于实现 |
| 是否可重复 | 可重复 | 不可重复 | key 不可重复 |
| 有无索引 | 有 | 无 | 无 |
| 是否键值对 | 否 | 否 | 是 |
| 顶层接口 | `Collection` | `Collection` | `Map`（独立） |

---

## 五、常用选择建议

| 需求 | 推荐集合 |
|------|----------|
| 需要有序、可重复、按索引操作 | `ArrayList` |
| 需要频繁增删 | `LinkedList` |
| 需要去重、不关心顺序 | `HashSet` |
| 需要去重且保持插入顺序 | `LinkedHashSet` |
| 需要去重且自动排序 | `TreeSet` |
| 需要键值对、不关心顺序 | `HashMap` |
| 需要键值对且保持插入顺序 | `LinkedHashMap` |
| 需要键值对且按 key 排序 | `TreeMap` |
