# Java 集合遍历方法详解

---

## 一、概述

Java 中遍历集合的方式主要有以下几种：

1. **普通 for 循环** —— 适用于有索引的集合（如 List）
2. **增强 for 循环（foreach）** —— 适用于所有 Collection 和数组
3. **迭代器（Iterator）** —— 适用于所有 Collection，可在遍历中安全删除
4. **Lambda 表达式（forEach）** —— JDK 8+，代码最简洁
5. **Stream 流** —— JDK 8+，支持链式操作（过滤、映射等）

下面逐一介绍每种方式的语法、特点和使用场景。

---

## 二、普通 for 循环（按索引遍历）

### 语法

```java
for (int i = 0; i < list.size(); i++) {
    E element = list.get(i);
    // 处理 element
}
```

### 特点

- 只能用于有索引的集合，如 `ArrayList`、`LinkedList`、`Vector`
- 不能用于 `Set`、`Queue`、`Map`（它们没有索引）
- 可以精确控制遍历的起始位置和步长
- 对于 `ArrayList` 效率高（数组随机访问 O(1)）
- 对于 `LinkedList` 效率低（链表需要从头遍历到目标位置 O(n)）

### 示例

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

for (int i = 0; i < list.size(); i++) {
    System.out.println("索引 " + i + ": " + list.get(i));
}
```

**输出：**

```
索引 0: A
索引 1: B
索引 2: C
```

## 三、增强 for 循环（foreach）

### 语法

```java
for (元素类型 变量名 : 集合或数组) {
    // 处理 变量名
}
```

### 原理

- 编译后实际上会被转换为**迭代器（Iterator）**方式
- 反编译后可以看到底层仍然是 `iterator()` + `hasNext()` + `next()`

### 特点

- 适用于所有实现了 `Iterable` 接口的集合（Collection 全部支持）
- 也适用于数组
- 语法简洁，不需要关心索引
- **不能在遍历过程中删除元素**（会抛 `ConcurrentModificationException`（并发修改异常））
- 不能获取当前元素的索引位置

### 示例 1 —— 遍历 List

```java
List<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
list.add("C++");

for (String s : list) {
    System.out.println(s);
}
```

**输出：**

```
Java
Python
C++
```

### 示例 2 —— 遍历 Set

```java
Set<Integer> set = new HashSet<>();
set.add(10);
set.add(20);
set.add(30);

for (Integer num : set) {
    System.out.println(num);
}
```

### 示例 3 —— 遍历数组

```java
int[] arr = {1, 2, 3, 4, 5};
for (int n : arr) {
    System.out.println(n);
}
```

### ⚠️ 注意

增强 for 循环中删除元素会抛异常：

```java
for (String s : list) {
    if ("Java".equals(s)) {
        list.remove(s);  // ❌ ConcurrentModificationException
    }
}
```

> 如需遍历中删除，请使用**迭代器的 `remove()` 方法**。

## 四、迭代器（Iterator）

### 语法

```java
Iterator<E> it = collection.iterator();
while (it.hasNext()) {
    E element = it.next();
    // 处理 element
    // it.remove();  // 如果需要删除当前元素
}
```

### 核心方法

| 方法 | 说明 |
|------|------|
| `hasNext()` | 判断是否还有下一个元素，返回 `boolean` |
| `next()` | 返回下一个元素，并将指针后移 |
| `remove()` | 删除当前迭代器指向的元素（可选操作） |
| `forEachRemaining(Consumer)` | JDK 8+，对剩余元素执行操作 |

### 特点

- 适用于所有 Collection 实现类
- 可以在遍历过程中安全删除元素（通过迭代器的 `remove()`）
- 单向遍历，只能向前
- 相比增强 for，代码稍多，但功能更强（支持删除）

### 示例 1 —— 基本遍历

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
```

### 示例 2 —— 遍历中安全删除元素

```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("C");

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("B".equals(s)) {
        it.remove();  // ✅ 安全删除，不会抛异常
    }
}
System.out.println(list);  // [A, C]
```

### 示例 3 —— forEachRemaining（JDK 8+）

```java
Iterator<String> it = list.iterator();
it.forEachRemaining(s -> System.out.println(s));
```

### ⚠️ 注意

- 每次调用 `next()` 前必须先调用 `hasNext()` 判断
- `remove()` 必须在 `next()` 之后调用，且每次 `next()` 只能调用一次 `remove()`
- **不要在迭代器外部直接使用集合的 `remove()`/`add()`**，否则会抛异常

## 五、Lambda 表达式（forEach 方法，JDK 8+）

### 语法

```java
collection.forEach(元素 -> {
    // 处理元素
});
```

或方法引用：

```java
collection.forEach(System.out::println);
```

### 原理

- `Iterable` 接口在 JDK 8 新增了默认方法 `forEach(Consumer action)`
- 内部仍然使用增强 for 循环实现（即底层仍是迭代器）

### 特点

- 代码极其简洁，一行即可完成遍历
- 适用于所有实现了 `Iterable` 接口的集合
- 不能在其中使用 `return` 跳出整个循环（`return` 只跳出当前 lambda）
- 不能在遍历过程中删除元素（与增强 for 一样会抛并发修改异常）
- 可以使用方法引用进一步提升可读性

### 示例 1 —— 基本 lambda 遍历

```java
List<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
list.add("Go");

list.forEach(s -> System.out.println(s));
```

### 示例 2 —— 方法引用（最简写法）

```java
list.forEach(System.out::println);
```

### 示例 3 —— 多条语句使用大括号

```java
list.forEach(s -> {
    s = s.toUpperCase();
    System.out.println(s);
});
```

### 示例 4 —— 遍历 Map

```java
Map<String, Integer> map = new HashMap<>();
map.put("Alice", 25);
map.put("Bob", 30);
map.put("Charlie", 35);

map.forEach((key, value) -> System.out.println(key + " = " + value));
```

### Lambda vs 增强 for

| Lambda forEach | 增强 for |
|---------------|----------|
| 代码更简洁 | 稍长一些 |
| 不能抛出受检异常 | 可以抛出受检异常 |
| 不能操作外部非 final 变量 | 可以 `break`/`continue` |

## 六、Stream 流（JDK 8+）

### 语法

```java
collection.stream()
          .中间操作(...)   // 过滤、映射、排序等
          .终端操作(...);  // forEach、collect 等
```

### 特点

- 支持函数式编程风格，链式调用
- 可以对数据进行过滤（`filter`）、映射（`map`）、排序（`sorted`）等操作后再遍历
- 可以并行处理（`parallelStream()`）
- 不会修改原集合
- **一个 Stream 只能被消费一次**

### 示例 1 —— 基本遍历

```java
List<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
list.add("C++");

list.stream().forEach(s -> System.out.println(s));
```

### 示例 2 —— 过滤后遍历

```java
list.stream()
    .filter(s -> s.startsWith("J"))  // 只保留以 J 开头的
    .forEach(System.out::println);    // 输出: Java
```

### 示例 3 —— 映射后遍历

```java
list.stream()
    .map(String::toUpperCase)  // 转为大写
    .forEach(System.out::println);
```

### 示例 4 —— 排序 + 过滤 + 遍历

```java
list.stream()
    .sorted()                         // 排序
    .filter(s -> s.length() > 2)      // 长度大于 2
    .forEach(System.out::println);    // 遍历输出
```

### 示例 5 —— 并行流遍历

```java
list.parallelStream().forEach(System.out::println);
```

## 七、遍历 Map 的几种方式

Map 不能直接遍历（没有实现 `Iterable`），需要通过中间视图来遍历。

### 方式 1 —— 遍历 keySet（遍历所有键）

```java
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);

for (String key : map.keySet()) {
    System.out.println(key + " = " + map.get(key));
}
```

### 方式 2 —— 遍历 values（遍历所有值）

```java
for (Integer value : map.values()) {
    System.out.println(value);
}
```

### 方式 3 —— 遍历 entrySet（推荐，一次获取键和值）

```java
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
```

### 方式 4 —— 迭代器遍历 entrySet

```java
Iterator<Map.Entry<String, Integer>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<String, Integer> entry = it.next();
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
```

### 方式 5 —— Lambda（最简洁）

```java
map.forEach((key, value) -> System.out.println(key + " = " + value));
```

### 方式 6 —— Stream 流

```java
map.entrySet().stream()
    .filter(e -> e.getValue() > 1)
    .forEach(e -> System.out.println(e.getKey() + " = " + e.getValue()));
```

## 八、各遍历方式对比总结

| 遍历方式 | 适用集合 | 支持删除 | 支持索引 | JDK 版本 | 简洁度 |
|----------|----------|----------|----------|----------|--------|
| 普通 for | List 系列 | ❌（有坑） | ✅ | 1.0 | ★★★ |
| 增强 for | Collection + 数组 | ❌ | ❌ | 5 | ★★★★★ |
| 迭代器 Iterator | Collection | ✅ | ❌ | 1.2 | ★★★ |
| Lambda forEach | Collection | ❌ | ❌ | 8 | ★★★★★ |
| Stream 流 | Collection | ❌ | ❌ | 8 | ★★★★ |

## 九、选择建议

### 1. 日常遍历，不需要删除元素 → 增强 for 循环 或 Lambda forEach

```java
list.forEach(System.out::println);
```

### 2. 需要中间操作（过滤、映射、排序）→ Stream 流

```java
list.stream().filter(...).map(...).forEach(...);
```

### 3. 需要在遍历中删除元素 → 迭代器 Iterator（或 removeIf）

```java
list.removeIf(s -> s.equals("B"));  // JDK 8+，一行搞定
```

### 4. 需要按索引操作 → 普通 for 循环（仅 List）

```java
for (int i = 0; i < list.size(); i++) { ... }
```
